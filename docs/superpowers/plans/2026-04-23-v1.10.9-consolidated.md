```python
def _safe_fetch_rfc822(mail, uid_str: str, attempts: int = 3, delay: float = 2.0):
    """FETCH RFC822 for a UID with retry + expunged-UID guard.

    Returns the raw message bytes, or None if the UID is no longer available
    (expunged between SEARCH and FETCH, flags-only response, or transport
    errors that don't recover).
    """
    import imaplib
    import ssl
    for attempt in range(1, attempts + 1):
        try:
            status, data = mail.uid('FETCH', uid_str, '(RFC822)')
        except (imaplib.IMAP4.abort, ssl.SSLError, OSError) as e:
            logger.warning(f"IMAP transport error on FETCH uid={uid_str} attempt {attempt}: {e}")
            if attempt < attempts:
                time.sleep(delay)
            continue
        if status != 'OK':
            if attempt < attempts:
                time.sleep(delay)
            continue
        if not data or data[0] is None or not isinstance(data[0], tuple) or len(data[0]) < 2:
            return None
        return data[0][1]
    return None
```

```python
def _expand_env(obj):
    """Recursively expand ${VAR_NAME}. Raises ValueError on unresolved vars."""
    if isinstance(obj, str):
        unresolved = []
        def _replace(m):
            name = m.group(1)
            val = os.environ.get(name)
            if val is None:
                unresolved.append(name)
                return m.group(0)
            return val
        result = re.sub(r'\$\{(\w+)\}', _replace, obj)
        if unresolved:
            raise ValueError(f"Unresolved environment variable(s): {', '.join(unresolved)}")
        return result
    if isinstance(obj, dict):
        return {k: _expand_env(v) for k, v in obj.items()}
    if isinstance(obj, list):
        return [_expand_env(i) for i in obj]
    return obj
```

## 1. Yandex IMAP account locked or rejecting login

**Investigate:**

2. Revoke old app password, generate new one.
3. Update `/home/adminos/email-processor/.env`:
   ```
   export IMAP_PASSWORD="new-app-password"
   ```
4. Restart cron-related shell or `source .env`.
5. Test: `python3 main.py --dry-run`.

## 2. Network share (/mnt/storage) unreachable

**Symptom:** Email report shows `Network share not reachable (timed out after 10s)`; daily CSV not written.

**Investigate:**

```bash
ls /mnt/storage
```

- SMB version mismatch (policy update). Look for `STATUS_LOGON_FAILURE` in dmesg → try:

  ```bash
  ls /mnt/storage
  ```
- Server reboot / credentials changed: verify with `sudo mount -a -v`.
- DNS / network: `ping 10.10.10.21`.

**Pipeline impact:** only CSV export is affected. Records are still written to local `master.xlsx`. Once the mount is back, next run auto-resumes the CSV export. You may need to manually backfill the CSV for the missed day.



**If cache claims a validity window ending in a past month:**
```bash
rm zetta_password.json
python3 main.py --dry-run
```

The next run will re-fetch the monthly password from IMAP. If the monthly password email has not yet arrived for this month, Zetta extraction will fail until it arrives — this is normal at the start of the month if Zetta sends the password late.

**If cache is current but ZIPs still fail:** Zetta may have rotated the password mid-month. Delete the cache and wait for the next monthly password email, or check for a Zetta notification email about the rotation.

## Out of scope for this plan (documented deferrals)

These findings were real but intentionally excluded. Address in future releases if they bite:

- **Sender spoofing protection (A1-C2, A2)** — requires SPF/DKIM/DMARC checking or an allowlist with exact-address match. Architectural; needs its own design session.
- **Clinic dedup by `id` not `name` (A1-C3)** — requires migrating historical master.xlsx records to new keys. Do before the first clinic rename.
- **`is_password_zip_email` substring match (A1-C4)** — replace with exact-address check. Low probability; bundle with sender-allowlist work.
- **D/M/Y vs M/D/Y date ambiguity (A1-H1)** — low probability given insurer locale. Monitor dedup stats for signals.
- **Timezone-aware datetime everywhere (A1-H6)** — large scope. Document VM-must-be-Moscow requirement in `SETUP.md` (done) and defer the refactor.
- **Leading zeros in policy numbers (A2-5.3)** — parser-level fix. Monitor production dedup rate first to see if it's actually a problem.
- **CSV lock coverage gap (A2-6.1)** — separate lock architecture. Not an issue today because we don't run two cron jobs. Revisit when scaling.
- **SQLite WAL mode (A2-6.2)** — enable if concurrent-run contention becomes visible in logs.
- **`_migrate_csv_header` atomicity (A1-M2)** — run once under lock on first deploy; code rewrite low priority.
- **`_quarantine` `.err` sidecar (A1-M3)** — nice-to-have for operator diagnostics.
- **`subject_keywords` + `skip_rules` broadness (A1-M4, M5)** — requires `allowed_senders` populated (remind ops at launch).
- **`_is_zetta_notification` regex (A1-M7)** — tighten if false positives appear.
- **Triple file reads per attachment (A1-M1)** — `@lru_cache` on header rows. Performance tweak only.
- **LibreOffice concurrent-conversion races (A1-M9)** — prevented by A1's pidfile lock if both runs are on main.py. Only relevant if you run two unrelated LibreOffice invocations.
- **Pre-flight master.xlsx integrity check** — nice-to-have startup fast-fail.
- **Backup rotation for `*.bak`** — `backup.sh` tar.gz already covers this use case.
- **`zetta_password.json` cache-age alert** — monitor externally for now.
- **Metrics export / time-series** — bigger architectural question.
- **`--dry-run-imap` mode** — nice-to-have for the first week. Skip for now.
- **Schema validation on parsed records** — use a dataclass or Pydantic model in a future release.
- **PII in `parsers/zetta.py` error dumps** — reduce log level if ops complains.
- **`.xlsx` magic-number validation before parse** — minor.
